B1. Problem Formulation 8 marks

(a) 3 marks Formulate this as a machine learning problem. State clearly: what is the target variable, what are the candidate input features, and what type of ML problem is this? Justify your choice of problem type.

Problem Type and Variables**

Problem Type: This is a Regression problem.

Target Variable: items_sold (a continuous numerical value).

Candidate Input Features:

Store Attributes: Store ID, Location Type (Urban/Rural), Store Size, Competition Density.

Promotion Details: Promotion Type (BOGO, Flat Discount, etc.).

Temporal Features: Month, Year, Festival Flags, Weekend Flags.

Justification: We are predicting a specific quantity (number of items) rather than a category. By predicting the volume for each promotion type, the business can select the one that yields the highest predicted value.



(b) 3 marks The company currently measures performance using total sales revenue. Explain why using items sold (sales volume) is a more reliable target variable for this problem. What broader principle does this illustrate about target variable selection in real-world ML projects?

Using items sold (volume) is more reliable than revenue because revenue is heavily influenced by the price of the items sold. A high-revenue month might simply mean a few expensive leather jackets were sold, whereas "items sold" directly measures the customer response and footfall generated by a promotion. This illustrates the principle that the target variable should be as "close" to the behavior you want to influence as possible, minimizing external noise like price fluctuations or inflation.



(c)2 marks A junior analyst suggests running one single global model across all 50 stores. Propose and justify an alternative modelling strategy that accounts for the fact that stores in different locations respond very differently to the same promotion.

Instead of a single global model, I propose a Clustered Modeling strategy. We can group stores into segments (e.g., a model specifically for Urban stores and another for Rural stores). Alternatively, we can use a Global Model with Entity Embeddings or "Categorical Encoding" for Store IDs and Location Types. This allows the model to learn specific "weights" or behaviors for different demographics rather than treating a rural shop the same as a high-street urban outlet.


B2. Data and EDA Strategy -- 10 marks

(a) 4 marks The raw data arrives in four separate tables: transactions, store attributes, promotion details, and a calendar (with weekend and festival flags). Describe how you would join these tables. What is the grain of the final modelling dataset (one row = what?), and what aggregations would you perform before modelling?

Joining Logic: I would perform a left join starting with the Calendar table, joining Transactions on Date, then joining Store Attributes on Store_ID, and finally Promotion Details on Promotion_ID.

Final Grain: The grain will be One row per Store per Month.

Aggregations: I would sum the items_sold and revenue per month and calculate the average footfall for that month to match the store-level monthly grain.

Promotion vs. Sales Volume (Boxplot): To see which promotions generally lead to higher sales across all stores.

(c)2 marks You notice that 80% of transactions in the dataset occurred without any promotion. Describe how this imbalance could affect your model and what steps you would take to address it.

If 80% of data has no promotion, the model may become biased toward "normal" sales days and fail to capture the "lift" caused by a promotion.

Solution: I would use Stratified Sampling to ensure promotions are well-represented in the training set or create a "baseline" feature that represents average sales without promotions to help the model learn the incremental gain.

B3. Model Evaluation and Deployment 12 marks

(a)4 marks You have monthly store-level data spanning three years across 50 stores. Describe how you would set up the train-test split. Why is a random split inappropriate here? Which evaluation metrics would you use, and how would you interpret each in the context of this business problem?

Split Strategy: Use a Time-Based Split (e.g., use the first 30 months for training and the last 6 months for testing).

Why not Random? Random splits cause "data leakage"—the model shouldn't know what happened in December 2025 to predict November 2025.

Metrics: * RMSE (Root Mean Squared Error): To penalize large prediction errors.

MAE (Mean Absolute Error): To understand the average "off-by" count in units sold, which is easy to explain to the marketing team.

(b)4 marks After training, the model recommends the Loyalty Points Bonus for Store 12 in December and the Flat Discount for Store 12 in March. Using the concept of feature importance, explain how you would investigate and communicate to the marketing team why the model makes different recommendations for the same store in different months.

If the model recommends different promotions for the same store in December vs. March, I would look at SHAP values or Feature Importance. In December, the is_festival feature likely has high importance, driving the "Loyalty Points" recommendation to capture holiday shoppers. In March, without a festival, the competition_density might become more influential, leading the model to suggest a "Flat Discount" to lure customers away from nearby competitors.

(c) 4 marks The trained model needs to generate recommendations at the start of every month for all 50 stores without being retrained each time. Describe the end-to-end deployment process: how you would save the model, how new monthly data would be prepared and fed in, and what monitoring you would put in place to detect when the model's performance has degraded and retraining is needed.

Deployment Save the model using joblib or pickle. Create a Python script that takes the upcoming month's "planned data" (Store ID, Month, Promotion Type) and outputs the predicted items sold.

Preparation: New monthly data would be fed through the same Preprocessing Pipeline (Scaling and One-Hot Encoding) built during training.

Monitoring: I would implement Model Drift monitoring. If the RMSE on real-world sales starts to increase significantly over 2-3 months, it indicates customer behavior has changed, and the model needs to be retrained on the most recent data.
